# Практические задания 21 и 22

## Задание 21: Кэширование запросов

Реализовано в проекте `practice-11-12-roles-api`.

### Реализация

Добавлено кэширование для следующих маршрутов с использованием пакета `node-cache`:

| Маршрут | Метод | Время кэша | Описание |
|---------|-------|------------|----------|
| `/api/users` | GET | 1 минута | Получить список пользователей |
| `/api/users/:id` | GET | 1 минута | Получить пользователя по id |
| `/api/products` | GET | 10 минут | Получить список товаров |
| `/api/products/:id` | GET | 10 минут | Получить товар по id |

### Ключевые особенности

- **Автоматическая инвалидация кэша** при изменении данных (POST, PUT, DELETE)
- **Разное время жизни** для разных типов данных
- **Прозрачная работа** — кэширование не влияет на API контракт

### Установка и запуск

```bash
cd practice-11-12-roles-api
npm install
npm start
```

### Проверка работы кэша

```bash
# Первый запрос (кэш заполняется)
curl http://localhost:4100/api/products

# Второй запрос (данные из кэша)
curl http://localhost:4100/api/products

# После изменения данных (кэш инвалидируется)
curl -X POST http://localhost:4100/api/products \
  -H "Content-Type: application/json" \
  -d '{"title":"Test","category":"Test","description":"Test","price":100}'
```

## Задание 22: Балансировка нагрузки

Реализовано в проекте `practice-21-22-load-balancing`.

### Структура проекта

```
practice-21-22-load-balancing/
├── backend/
│   └── server.js          # Backend-сервер Express
├── nginx/
│   └── nginx.conf         # Конфигурация Nginx
├── haproxy/
│   └── haproxy.cfg        # Конфигурация HAProxy
└── package.json
```

### Компоненты

#### Backend-серверы

Два идентичных сервера, работающих на разных портах:
- **Backend 1**: порт 3001
- **Backend 2**: порт 3002

Каждый сервер возвращает уникальный `server_id` для отслеживания распределения запросов.

#### Nginx Load Balancer

Поддерживает три алгоритма балансировки:

| Порт | Алгоритм | Описание |
|------|----------|----------|
| 80 | round-robin | По умолчанию, циклическое распределение |
| 8081 | least_conn | Наименьшее количество соединений |
| 8082 | ip_hash | Фиксированный сервер для одного IP |

**Настройки отказоустойчивости:**
- `max_fails=3` — исключить сервер после 3 неудач
- `fail_timeout=30s` — время исключения сервера

#### HAProxy Load Balancer

Поддерживает алгоритмы балансировки:

| Порт | Алгоритм | Описание |
|------|----------|----------|
| 8080 | roundrobin | По умолчанию |
| 8083 | leastconn | Наименьшее количество соединений |
| 8084 | source | По IP клиента |

**Настройки отказоустойчивости:**
- `inter 5000` — проверка каждые 5 секунд
- `fall 3` — исключить после 3 неудачных проверок
- `rise 2` — вернуть после 2 успешных проверок

**Statistics Dashboard:** http://localhost:8404/

### Установка и запуск

```bash
cd practice-21-22-load-balancing
npm install
```

#### Запуск backend-серверов

```bash
# Запуск обоих серверов одновременно
npm run start:all

# Или по отдельности
npm run start:backend1  # порт 3001
npm run start:backend2  # порт 3002
```

#### Запуск Nginx

```bash
# Проверка конфигурации
nginx -t -c /path/to/nginx/nginx.conf

# Запуск
nginx -c /path/to/nginx/nginx.conf

# Остановка
nginx -s stop
```

#### Запуск HAProxy

```bash
# Запуск
haproxy -f /path/to/haproxy/haproxy.cfg

# Или в фоне
haproxy -f /path/to/haproxy/haproxy.cfg -D
```

### Тестирование балансировки

#### Проверка распределения запросов (Nginx)

```bash
# Отправить 10 запросов и посмотреть распределение
for i in {1..10}; do
  curl -s http://localhost:80 | jq '.server_id'
done
```

#### Проверка распределения запросов (HAProxy)

```bash
# Отправить 10 запросов и посмотреть распределение
for i in {1..10}; do
  curl -s http://localhost:8080 | jq '.server_id'
done
```

#### Проверка health endpoint

```bash
# Nginx
curl http://localhost:80/health

# HAProxy
curl http://localhost:8080/health
```

#### Тестирование отказоустойчивости

1. Запустить оба backend-сервера
2. Отправить несколько запросов для подтверждения работы
3. Остановить один из серверов
4. Отправить ещё запросов — балансировщик должен перенаправлять трафик на рабочий сервер
5. Запустить остановленный сервер — он должен автоматически вернуться в пул

### Сравнение Nginx и HAProxy

| Характеристика | Nginx | HAProxy |
|----------------|-------|---------|
| Алгоритмы балансировки | round-robin, least_conn, ip_hash | roundrobin, leastconn, source |
| Health checks | ✓ (пассивные) | ✓ (активные) |
| Статистика | Базовая | Расширенная (веб-интерфейс) |
| SSL/TLS | ✓ | ✓ |
| Кэширование | ✓ | ✗ |
| Reverse proxy | ✓ | ✓ |
